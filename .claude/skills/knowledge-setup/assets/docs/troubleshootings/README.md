# Troubleshootings

개발 중 발생한 **이슈·개발 실패·VOC**를 기록한다. 같은 문제가 반복되는 것을 막는 것이 목적이므로,
증상으로 검색이 가능하도록 작성하고 **원인과 해결**을 분명히 남긴다.

## 인덱스

| 파일 | 한 줄 요약(증상) | 갱신일 |
|---|---|---|
| _(아직 항목 없음)_ | | |

## 항목 작성 템플릿

새 항목은 아래 형식으로 `<증상-요약>.md` 파일을 만들어 작성하고, 위 인덱스 표에 한 줄을 추가한다.

```markdown
# <증상 요약 제목>

- 날짜: YYYY-MM-DD
- 환경/범위: (예: 런타임 버전 / 특정 환경 / CI 등 해당 시)

## 증상
무엇이 어떻게 잘못됐는가. 에러 메시지·로그를 그대로 인용하면 검색에 유리.

## 원인
근본 원인. 추측이 아니라 확인된 내용.

## 해결
어떻게 해결했는가. 적용한 변경/명령/설정.

## 재발 방지 / 참고
다시 겪지 않으려면 무엇을 주의해야 하는가. 관련 링크·이슈·커밋.
```

## 네이밍 규칙

kebab-case, 증상을 요약. 예: `build-fails-after-dependency-upgrade.md`, `flaky-test-on-ci.md`
