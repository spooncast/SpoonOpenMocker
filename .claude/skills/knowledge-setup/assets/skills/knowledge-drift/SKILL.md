# knowledge-drift

`.claude/docs/`의 각 카테고리에서 **인덱스 표**와 **실제 항목 파일**이 어긋나지 않았는지
점검한다. 지식 베이스가 커질수록 "파일은 추가했는데 인덱스에 안 올렸다"거나 그 반대 같은
불일치가 쌓이는데, 이 스킬이 그것을 찾아낸다.

## 점검 항목

- **고아 파일**: 폴더에는 있으나 인덱스 표에 없는 항목 파일.
- **깨진 링크**: 인덱스 표에는 있으나 실제로 없는 파일.
- **네이밍 위반**: kebab-case(소문자·숫자·하이픈)가 아닌 파일명.

## 절차

1. 점검 스크립트를 실행한다(저장소 루트에서):

   ```bash
   python3 .claude/skills/knowledge-drift/scripts/check_drift.py
   ```

   특정 경로만 보려면 인자로 docs 경로를 넘긴다:
   `python3 .claude/skills/knowledge-drift/scripts/check_drift.py .claude/docs`

   스크립트는 README.md가 있는 모든 하위 카테고리를 자동으로 순회하므로, 나중에 추가된
   카테고리도 별도 설정 없이 함께 점검된다. 종료 코드는 문제가 없으면 0, 하나라도 있으면
   1이다(나중에 pre-commit 훅이나 CI에 그대로 연결할 수 있다).

2. 출력 결과를 사용자에게 카테고리별로 정리해 전달한다. 문제가 없으면 "정합성 이상 없음"으로
   간단히 보고한다.

3. 문제가 있으면 각 항목의 **고치는 방법**을 함께 제시한다:
   - 고아 파일 → 해당 카테고리 인덱스 표에 행을 추가(`knowledge-update`의 인덱스 갱신 절차).
   - 깨진 링크 → 파일이 옮겨졌는지/이름이 바뀌었는지 확인 후 인덱스 행을 수정하거나 제거.
   - 네이밍 위반 → kebab-case로 파일명을 바꾸고 인덱스의 링크도 함께 수정.

   **수정은 사용자 확인 후 진행한다.** 깨진 링크나 이름 변경은 의도된 삭제/이동일 수 있어
   자동으로 손대면 위험하다. 무엇을 어떻게 고칠지 제안하고 동의를 받은 뒤 적용한다.

## 참고

점검은 `## 인덱스` 섹션의 표를 기준으로 한다. 항목 추가 시 인덱스 형식을 일관되게 유지하려면
`knowledge-update` 스킬을 사용하라. 두 스킬은 짝을 이룬다 — 하나는 올바르게 기록하고, 하나는
어긋난 곳을 찾는다.
