# Create Branch

베이스 브랜치에서 지라 티켓명으로 새 브랜치를 생성하고 체크아웃하는 스킬.

## 필요한 인자

1. **베이스 브랜치명** — 분기의 기준이 되는 브랜치 (예: `release/11.3.0`, `develop`, `main`)
2. **지라 티켓명** — 새로 생성할 브랜치 이름 (예: `AOSTEAM-1234`)

## 사용법

번들된 스크립트를 실행한다:

```bash
python3 <skill-directory>/scripts/create_branch.py <베이스_브랜치> <지라_티켓명>
```

**예시:**
```bash
python3 <skill-directory>/scripts/create_branch.py release/11.3.0 AOSTEAM-1234
```

## 베이스 브랜치 결정

- **대화 중에 베이스 브랜치를 파악했으면, 그 베이스 브랜치에서 새 브랜치를 생성한다.** (예: 직전 대화에서 `release/11.11.0`에서 작업한다고 했으면 그 브랜치를 베이스로 쓴다.)
- 대화에서 베이스 브랜치를 파악하기 어려우면 사용자에게 베이스 브랜치를 물어본다. 임의로 `develop`/`main` 등을 가정하지 않는다.

## 동작 순서

1. `git fetch origin <베이스_브랜치>` — 원격의 베이스 브랜치를 최신 상태로 가져온다
2. `git checkout -b <지라_티켓명> origin/<베이스_브랜치>` — 베이스 브랜치 기준으로 새 브랜치를 생성하고 체크아웃한다

## 주의사항

- Git 저장소 내에서 실행해야 한다. 저장소 밖에서 실행하면 오류가 발생한다.
- 이미 동일한 이름의 브랜치가 존재하면 git이 오류를 반환한다. 이 경우 사용자에게 알려주고 기존 브랜치를 체크아웃할지 확인한다.
- 원격에 해당 베이스 브랜치가 없으면 fetch가 실패한다. 브랜치명을 다시 확인하도록 안내한다.

## 인자가 누락된 경우

두 인자(베이스 브랜치, 지라 티켓명) 중 누락된 것을 채운다.

- **베이스 브랜치**: 대화 맥락에서 파악했으면 그대로 쓰고, 파악이 어려우면 사용자에게 물어본다.
- **지라 티켓명**: 대화 맥락(직전에 만든 티켓 등)에서 파악했으면 그대로 쓰고, 없으면 물어본다.
