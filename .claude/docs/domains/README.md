# Domains

프로젝트에서만 통용되는 **도메인 용어**를 정의한다. 신규 합류자나 AI가 코드/대화 속 용어를
정확히 이해하도록 돕는다. 일반적인 기술 용어가 아니라 이 프로젝트 고유의 의미를 가진 것만 기록한다.

## 인덱스

| 파일 | 용어 | 한 줄 정의 | 갱신일 |
|---|---|---|---|
| _(아직 항목 없음)_ | | | |

## 항목 작성 템플릿

새 항목은 아래 형식으로 `<용어>.md` 파일을 만들어 작성하고, 위 인덱스 표에 한 줄을 추가한다.

```markdown
# <용어>

## 정의
이 프로젝트에서 이 용어가 정확히 무엇을 가리키는가.

## 사용 맥락 / 위치
어디서(화면/모듈/코드/문서) 어떻게 쓰이는가. 가능하면 코드 경로 예시.

## 관련 용어
함께 이해해야 할 다른 용어 (다른 domains 항목 링크).
```

## 네이밍 규칙

kebab-case, 용어 자체. 예: `aggregate-root.md`, `billing-cycle.md`
