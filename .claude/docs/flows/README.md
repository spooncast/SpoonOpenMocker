# Flows

특정 기능을 추가/변경할 때의 **개발 순서와 방향**을 기록한다.
(예: API 추가 시 data layer → domain layer → presentation layer 순으로 개발한다.)
반복되는 작업을 정해진 절차대로 진행해 누락과 시행착오를 줄인다.

## 인덱스

| 파일 | 한 줄 요약 | 적용 시점 | 갱신일 |
|---|---|---|---|
| _(아직 항목 없음)_ | | | |

## 항목 작성 템플릿

새 항목은 아래 형식으로 `<플로우>.md` 파일을 만들어 작성하고, 위 인덱스 표에 한 줄을 추가한다.

```markdown
# <플로우 제목>

- 적용 시점: 이 절차를 따라야 하는 상황 (예: 새 API 연동을 추가할 때)

## 단계
1. 1단계 — 무엇을, 어디에
2. 2단계 — ...
3. 3단계 — ...

## 체크리스트
- [ ] 확인할 항목 1
- [ ] 확인할 항목 2
```

## 네이밍 규칙

kebab-case, 플로우 주제. 예: `add-api-endpoint.md`, `add-ui-screen.md`
