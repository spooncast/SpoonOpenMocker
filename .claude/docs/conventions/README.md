# Conventions

프로젝트의 **컨벤션**을 기록한다. (예: 네이밍 규칙, 레이어 의존성 규칙, 테스트 작성 규칙)
코드를 작성·리뷰할 때 따라야 할 규칙을 모아 일관성을 유지한다.

## 인덱스

| 파일 | 한 줄 요약 | 적용 범위 | 갱신일 |
|---|---|---|---|
| _(아직 항목 없음)_ | | | |

## 항목 작성 템플릿

새 항목은 아래 형식으로 `<주제>.md` 파일을 만들어 작성하고, 위 인덱스 표에 한 줄을 추가한다.

```markdown
# <컨벤션 제목>

- 적용 범위: (예: 전체 코드 / 특정 모듈·레이어 / 테스트 등)

## 규칙
- 규칙 1
- 규칙 2

## 예시
권장:
\`\`\`
// 권장 예시
\`\`\`
지양:
\`\`\`
// 지양 예시
\`\`\`

## 예외
규칙을 적용하지 않는 경우와 그 이유.
```

## 네이밍 규칙

kebab-case, 컨벤션 주제. 예: `naming-conventions.md`, `unit-test-conventions.md`, `api-error-format.md`
